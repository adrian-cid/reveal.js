#1. Core UI

  • Pros
  • "Easy" to use.
  • No dev skills needed.
  • Changes are quick.
  • Cons
  • Limited capabilities.
  • Options too broad.
  • Options too narrow.

tl;dr Out of the box functionality.

#2. Modules

  • Pros
  • Extends Drupal core.
  • Many options available.
  • None exist? Build your own.
  • Cons
  • Added complexity.
  • May do more then needed.
  • Potential Security issues.

tl;dr Extend Drupal core.

#3. Templates

  • Pros
  • Replace Drupal templates.
  • Hardcode or use Twig logic.
  • Changes are cached.
  • Cons
  • Templates may be wrong level.
  • Not everything is templated.
  • May require server side code.

tl;dr Control rendered HTML.

  {% if items %}
    {% if menu_level == 0 %}
      <ul{{ attributes.addClass('menu') }}>
    {% else %}

    {% endif %}

      {# ... #}

{% endmacro %}

#4. Custom CSS

  • Pros
  • Control of display.
  • Target specific elements.
  • Can be scripted with SASS.
  • Cons
  • Can only change display.
  • Complex element chains.
  • ??

tl;dr Position, size and color.

#5. Custom JavaScript

  • Pros
  • Manipulate any DOM item.
  • Can be based on actions.
  • Less server processing.
  • Cons
  • Browser dependencies.
  • Delay before execution.
  • Alterable by client.

tl;dr Client controlled functionality.

#6. Hooks

  • Pros
  • Provided by core and modules.
  • Use in custom theme or module.
  • Easiest way to extend core.
  • Cons
  • Not used everywhere.
  • May be wrong level.
  • May have wrong timing.

tl;dr Core manipulation functions.
